Noteworthy

  1. Utilized the Bulma framework to make the app mobile friendly. The framework was used to give color and spacing to many elements of each page. The select lists and buttons were customized with Bulma to give them more style.
  2. The JavaScript code is written in ES6 modules which imports and exports appriopriate data.
  3. Each page has a header and a footer that were created using web components. Another component was the tips section within the "App" page.
  4. Data about the enemies in the game were loaded in from a JSON file using fetch.
  5. The code originated from a homework assignment which was a clicker game. The image drawing and movement code were utilzed and enhanced to give it more functionality and feedback.
  6. The game has a high score system where it updates and saves the user's top 5 scores to the local storage. At the end of the game, the user's final score is compared with an array in the storage. If it is higher than any of the scores, it will replace the score within the leaderboard.
  7. The game uses canvas to draw and play.
  8. The header will let the user know what page they are on.
  9. The game contains background music and various sound effects to enhance the player experience and to provide feedback for certain actions.
  10. The game contains special abilities activated by key presses. Using these abilities at the right time can allow the player to beat levels within a shorter amount of time. If the player is not satisfied with their current progress, they have the ability to restart the level to get a better score.
  11. The levels were designed to help the player learn how to play the game. The first level is slow paced which helps the player to get used to the controls. Subsequent levels are sped up a little with more enemies to make it more challenging.

Grading

  1. The app contains at least four controls being mouse clicks and three different key presses.
  2. Instructions are listed in various different sections in the app. A tips section is always displayed next to the game in case the user forgets how to use an ability while playing.
  3. The game saves the user's top five highest scores to the local storage.
  4. Data about enemies are loaded in from a JSON file.
  5. The navbar will highlight and bold the current page that the user is on.
  6. Contains at least three web components.
  7. Uses ES6 modules and multiple embedded fonts.
  8. Various text were written in each of the game states to let the users know what state they are in and what they may need to do.
  9. The game contains multiple levels, each with progressing difficulty but is never impossible to beat.
  10. The game contains a bit of randomness whenever the bullet is split up as the projectiles can travel in random directions.
  11. Skill based gameplay can come from the skills. Depending on when the player uses the skills, it can give them extra momentum or save them from a pickle. The player has a cooldown before they can fire another projectile. This is especially important to know within late levels that contains multiple enemies who can fire at the player from multiple directions.
